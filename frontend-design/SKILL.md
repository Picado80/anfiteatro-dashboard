## Overview

This skill helps you design professional enterprise software solutions for medium-sized companies. It addresses real organizational challenges: lack of structure, poor visibility into metrics, scattered employee information, and no unified systems for management. Generate production-ready code, visual mockups, and design specifications that solve actual business problems.

## What This Skill Does

**Performance Management:**
- Executive dashboards showing company metrics, KPIs, and trends
- Employee performance tracking with individual metrics and progress
- Manager tools to monitor team performance over time
- Data visualization for decision-making

**HR & Employee Management:**
- Employee directories with searchable profiles and contact info
- Digital employee files (centralized, organized, accessible)
- Team structure and reporting hierarchy
- Succession planning and talent management

**Administrative Panels:**
- Resource management systems
- Department and team administration
- Goal and objective tracking
- Evaluation and review workflows

**Data Systems:**
- Organized data storage and retrieval
- Reporting and analytics
- Integration with existing systems (databases, APIs)
- Audit trails and compliance

## How to Use It

Be specific about the business problem you're solving:

**Performance Management Example:**
"Design a performance dashboard where HR and managers can see all employees, their KPI scores, progress over quarters, and identify high/medium/low performers. Include filters by department and manager."

**Employee Directory Example:**
"Create an employee directory app with searchable profiles showing name, department, role, contact info, hire date, and file attachments. Managers should be able to see their team, HR should see everyone."

**Admin Panel Example:**
"Design an HR admin panel to manage KPIs and goals for each employee. Include ability to set targets, track progress, and export reports."

## Output Formats

### 1. Production React Code
- Clean, reusable components with Tailwind CSS
- TypeScript for type safety
- Ready to integrate with real databases and APIs
- Proper state management and performance optimization

### 2. Data Structure Documentation
- How data should be organized in database
- API endpoints needed
- Data relationships and schemas
- Example data format

### 3. Visual Mockups
- Screenshots of desktop and mobile views
- State variations (loading, empty, error states)
- Interaction flows

### 4. Design Specifications
- Color system and usage

- Component library documentation
- Accessibility requirements

## Design Principles

**Enterprise Professional:**
- Clean, serious aesthetics (not trendy or playful)
- Proper information hierarchy and dense information when needed
- Color-coded status (green=good, yellow=attention, red=critical)
- Dark mode support for all-day usage

**User-Focused:**
- Managers need quick visibility into team performance
- HR needs comprehensive, searchable data
- Executives need high-level trends and insights
- Employees need to understand their own metrics

**Data-Driven:**
- Charts and visualizations that tell stories
- Sortable/filterable tables for large datasets
- Real-time updates where applicable
- Export capabilities for reporting

**Structured:**
- Consistent navigation and patterns
- Clear workflows for common tasks
- Organized information architecture
- Search and filtering for large datasets

## Implementation Notes

- Code works with Next.js, React, or vanilla setup
- Compatible with PostgreSQL, MongoDB, or any backend
- Can integrate with Google Sheets, Excel, or databases
- Supports user roles and permissions
- Mobile-responsive design
- Dark mode built-in
